⚠️ the problem

veyon sees everything

veyon is classroom monitoring software that lets teachers view and control student screens in real-time. it runs as a system service with deep access.

no privacy controls

once veyon is installed, the teacher can see your screen at any time without warning. there's no built-in way to know when you're being watched.

the network path

veyon uses a two-tier architecture: an external proxy receives connections from the teacher, then forwards to an internal VNC server.

the opportunity

if we can sit between those two components, we can intercept and modify what the teacher sees without them knowing anything changed.

🎛️ features

freeze mode
window hiding
presence overlay

freeze mode

press Ctrl+Alt+F to capture your current screen. from that moment on, veyoff sends the frozen frame to the teacher while you continue working normally.

// freeze logic if (frozen && frozenFrame.valid()) { sendFrame(frozenFrame); // send old frame } else { forwardToRealVnc(); // normal flow }

window hiding

blacklisted windows are painted black before being sent. veyoff enumerates visible windows, checks titles against keywords in blacklist.txt, and blacks out matching regions.

presence overlay

two-level warning system invisible to veyon:

📋 how it works step by step

🔍

1. registry read

reads VncServerPort from HKLM\SOFTWARE\Veyon Solutions\Veyon\Network

📝

2. port redirect

writes port + 50 to registry (e.g., 11200 to 11250)

🔄

3. service restart

restarts VeyonService to apply changes (1-2 second blip)

🎭

4. proxy bind

veyoff binds to original VNC port and accepts connections

🤝

5. RFB handshake

transparently forwards handshake, learns framebuffer size

⚡

6. intercept mode

parses RFB messages and can inject frozen/redacted frames
